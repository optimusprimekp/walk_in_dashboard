2. Candidate Check-In
When an already registered candidate arrives for the interview:

- Candidate enters Mobile Number / Registration ID / Token Number.
- System fetches candidate details automatically.
- DO NOT allow manual text entry for Department or Position.
- Department and Position must be selected only from predefined Master Data tables.

3. Master Data
Create master tables:

Departments
- id
- department_name
- status

Positions
- id
- department_id
- position_name
- status

Interview Panels
- id
- panel_name
- department_id
- position_id
- interviewer_id
- room_number
- status

4. Check-In Form
Show dropdowns:

Department *
Position *

Rules:
- Department dropdown loads active departments.
- Position dropdown loads positions belonging to selected department.
- No free-text input allowed.
- Only dropdown selection.

5. Auto Queue Assignment
After candidate selects Department and Position and clicks Check-In:

System should automatically:
- Find active interview panel matching Department + Position.
- Assign candidate to that panel queue.
- Generate Queue Number.
- Status = "Waiting".
- Add candidate to live waiting queue of that panel.

Example:

Candidate:
Department = IT
Position = React Developer

System automatically assigns candidate to:
Panel 3 (IT + React Developer)

Candidate should immediately appear in:
Interviewer Queue Screen → Panel 3 Waiting List

No manual assignment required.

6. Interview Queue Logic

Status Flow:

Registered
→ Checked-In
→ Waiting
→ Called
→ In Interview
→ Buffer Time (5 min)
→ Completed

Interview Duration:
- Default 20 minutes
- Configurable per panel

Buffer Duration:
- Default 5 minutes
- Configurable per panel
